Introducción a Git y GitHub

Git es un sistema de control de versiones que permite a los desarrolladores colaborar en proyectos. GitHub es una plataforma que aloja repositorios Git y facilita la colaboración.

¿Por qué usar Git?

  • Guarda todo el historial de cambios
  • Permite trabajar en equipo
  • Facilita la recuperación ante errores
  • Permite trabajar en diferentes ramas

Flujo de trabajo básico con Git

  1. Inicializar: Crear un repositorio o clonar uno existente
  2. Modificar: Hacer cambios en los archivos
  3. Preparar: Seleccionar cambios para el próximo commit
  4. Confirmar: Guardar los cambios en el historial
  5. Sincronizar: Compartir cambios con el repositorio remoto

Áreas principales en Git

Área Descripción
Working Directory Tu espacio de trabajo local
Staging Area Área donde preparas los cambios para el próximo commit
Local Repository Registro local de todos los commits
Remote Repository Versión del repositorio en GitHub

Comandos básicos de Git

Configuración inicial

# Configurar nombre de usuario
git config --global user.name "Tu Nombre"

# Configurar correo electrónico
git config --global user.email "tuemail@ejemplo.com"

Crear y clonar repositorios

# Inicializar un nuevo repositorio
git init

# Clonar un repositorio existente
git clone https://github.com/usuario/repositorio.git

Gestión básica de cambios

Comando Descripción
git status Muestra el estado actual del repositorio
git add Añade cambios al área de preparación
git commit Confirma los cambios preparados
git log Muestra el historial de commits

Sincronización con repositorios remotos

# Añadir un repositorio remoto
git remote add origin https://github.com/usuario/repositorio.git

# Subir cambios al repositorio remoto
git push origin main

# Traer cambios del repositorio remoto
git pull origin main

Trabajo con ramas en Git

Las ramas son una característica poderosa de Git que permite desarrollo paralelo.

Comandos para trabajar con ramas

  • Listar ramas: git branch
  • Crear rama: git branch nombre-rama
  • Cambiar de rama: git checkout nombre-rama
  • Crear y cambiar: git checkout -b nombre-rama
  • Fusionar ramas: git merge nombre-rama
